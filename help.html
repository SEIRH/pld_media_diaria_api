API PLD Média Diária

API intermediária que consolida os dados de PLD médio diário da CCEE (Câmara de Comercialização de Energia Elétrica) de múltiplos anos em um único endpoint, com cache automático de 24 horas.

https://pld-media-diaria-api.onrender.com

Visão Geral

A CCEE disponibiliza os dados de PLD segmentados por ano, cada um com um resource_id diferente. Esta API unifica todos os anos disponíveis em um único endpoint, tornando o consumo direto pelo Power BI simples e sem necessidade de transformações manuais.

Cache automático: os dados são coletados da CCEE uma vez por dia. O primeiro acesso após a expiração do cache pode levar entre 30–60 segundos. Todos os acessos seguintes são instantâneos.

Base URL

https://pld-media-diaria-api.onrender.com

Todos os endpoints são relativos a esta URL base. A API não requer autenticação.

Formato de Resposta

Os endpoints retornam CSV (padrão, separador ;, encoding UTF-8) ou JSON, conforme indicado em cada endpoint. O encoding UTF-8 com BOM (utf-8-sig) é usado nos downloads para compatibilidade com Excel e Power BI.

Endpoints

GET /pld Retorna todo o dataset consolidado ▶

Retorna o CSV completo com todos os anos disponíveis concatenados. Este é o endpoint principal para uso no Power BI.

Exemplo de requisição

GET https://pld-media-diaria-api.onrender.com/pld

Resposta

200 OK — text/plain; charset=utf-8
MES_REFERENCIA;SUBMERCADO;DIA;PLD_MEDIA_DIA;ano_referencia_api
202101;NORDESTE;08/01/2021;312.49;2021
202101;SUDESTE;08/01/2021;192.35;2021
...
GET /filtrar Filtra por coluna e valor ▶

Permite filtrar os dados por qualquer coluna e valor, selecionar apenas colunas específicas e escolher entre CSV (com download) ou JSON.

Parâmetros

ParâmetroTipoObrig.Descrição
coluna string Sim Nome da coluna a filtrar. Use /colunas para ver os nomes disponíveis.
valor string Sim Valor a buscar na coluna. A comparação é feita como string, sem distinção de espaços nas bordas.
colunas string Não Colunas a retornar, separadas por vírgula. Omita para retornar todas.
formato string Não Formato de retorno: csv (padrão, faz download) ou json.

Exemplos

Filtrar todos os registros do submercado NORDESTE, retornando apenas dia e PLD:

GET /filtrar?coluna=SUBMERCADO&valor=NORDESTE&colunas=DIA,PLD_MEDIA_DIA&formato=csv

Filtrar por submercado e retornar como JSON (útil para testes no browser):

GET /filtrar?coluna=SUBMERCADO&valor=SUDESTE&formato=json

Filtrar um ano específico:

GET /filtrar?coluna=ano_referencia_api&valor=2024

Filtrar por mês de referência:

GET /filtrar?coluna=MES_REFERENCIA&valor=202401

Resposta CSV

200 OK — text/csv · arquivo: ccee_SUBMERCADO_NORDESTE.csv
DIA;PLD_MEDIA_DIA
08/01/2021;312.49
09/01/2021;312.49
...

Resposta JSON

200 OK — application/json
[
  { "MES_REFERENCIA": "202101", "SUBMERCADO": "NORDESTE", "DIA": "08/01/2021", "PLD_MEDIA_DIA": "312.49" },
  { "MES_REFERENCIA": "202101", "SUBMERCADO": "NORDESTE", "DIA": "09/01/2021", "PLD_MEDIA_DIA": "312.49" },
  ...
]

Erros possíveis

400 Bad Request → Coluna 'xyz' não encontrada. Disponíveis: MES_REFERENCIA, SUBMERCADO, ...
400 Bad Request → Colunas inválidas: xyz. Disponíveis: MES_REFERENCIA, SUBMERCADO, ...
GET /colunas Lista as colunas disponíveis ▶

Retorna a lista de todas as colunas presentes no dataset consolidado. Útil para consultar os nomes exatos antes de usar o /filtrar.

Exemplo de requisição

GET https://pld-media-diaria-api.onrender.com/colunas

Resposta

200 OK — application/json
{
  "colunas": [
    "MES_REFERENCIA",
    "SUBMERCADO",
    "DIA",
    "PLD_MEDIA_DIA",
    "ano_referencia_api"
  ]
}
GET /status Verifica o estado do cache ▶

Retorna informações sobre o cache atual: se está válido, quando foi atualizado, quantos registros há e se ocorreram erros na última coleta.

Exemplo de requisição

GET https://pld-media-diaria-api.onrender.com/status

Resposta

200 OK — application/json
{
  "cache_valido": true,
  "ultima_atualizacao": "2026-03-04T08:32:11.443210",
  "expira_em": "2026-03-05T08:32:11.443210",
  "total_registros": 14832,
  "anos_coletados": [2021, 2022, 2023, 2024, 2025, 2026],
  "erros_na_ultima_coleta": []
}
GET /atualizar Força recoleta imediata ▶

Ignora o cache atual e dispara uma nova coleta completa dos dados da CCEE. Use quando a CCEE publicar dados novos e você não quiser aguardar as 24h do cache.

⚠️ Este endpoint pode levar 30–60 segundos para responder, pois realiza o scraping completo com pausas para evitar bloqueio.

Exemplo de requisição

GET https://pld-media-diaria-api.onrender.com/atualizar

Resposta

200 OK — application/json
{
  "mensagem": "Dados recoletados com sucesso.",
  "total_registros": 14832,
  "anos_coletados": [2021, 2022, 2023, 2024, 2025, 2026],
  "erros": []
}

Integração Power BI

O Power BI consome a API via Web.Contents no Power Query (Editor M). Siga os passos abaixo:

1

No Power BI Desktop, clique em Obter Dados → Consulta em Branco.

2

Clique em Editor Avançado e cole o código abaixo.

3

Clique em Concluído. O Power BI carregará e tipará os dados automaticamente.

Código Power Query — dataset completo (/pld)

let
    Fonte       = Web.Contents("https://pld-media-diaria-api.onrender.com/pld"),
    ConteudoCSV = Text.FromBinary(Fonte),
    CSV = Csv.Document(
        Text.ToBinary(ConteudoCSV, TextEncoding.Utf8),
        [
            Delimiter  = ";",
            Encoding   = TextEncoding.Utf8,
            QuoteStyle = QuoteStyle.None,
            Columns    = null
        ]
    ),
    ComCabecalho = Table.PromoteHeaders(CSV, [PromoteAllScalars = true])
in
    ComCabecalho

Código Power Query — com filtro (/filtrar)

Exemplo: buscar apenas o submercado NORDESTE com dia e PLD.

let
    Url = "https://pld-media-diaria-api.onrender.com/filtrar"
        & "?coluna=SUBMERCADO&valor=NORDESTE"
        & "&colunas=DIA,PLD_MEDIA_DIA",
    Fonte       = Web.Contents(Url),
    ConteudoCSV = Text.FromBinary(Fonte),
    CSV = Csv.Document(
        Text.ToBinary(ConteudoCSV, TextEncoding.Utf8),
        [Delimiter = ";", Encoding = TextEncoding.Utf8]
    ),
    ComCabecalho = Table.PromoteHeaders(CSV, [PromoteAllScalars = true])
in
    ComCabecalho
Dica: após carregar, use Transformar → Detectar Tipo de Dados para o Power BI tipar automaticamente datas e números.

Manutenção Anual

Como a CCEE separa os dados por ano e usa IDs únicos por dataset, é necessário adicionar o novo resource_id no início de cada ano.

Como encontrar o novo ID

1

Acesse dadosabertos.ccee.org.br e localize o dataset do novo ano.

2

Clique em Explorar no dataset. O resource_id aparece na URL ou na chamada de API da página.

3

Edite o dicionário RESOURCE_IDS_POR_ANO no main.py e faça o redeploy.

# main.py
RESOURCE_IDS_POR_ANO: dict[int, str] = {
    ...
    2026: "3ca83769-de89-4dc5-84a7-0128167b594d",
    2027: "cole-aqui-o-novo-resource-id",  # ← adicionar
}

Códigos de Erro

CódigoSituação
200 Sucesso.
400 Parâmetro inválido no /filtrar — coluna inexistente ou nome de coluna errado. A resposta inclui a lista de colunas válidas.
503 Nenhum dado disponível no cache — a coleta da CCEE falhou para todos os anos. Verifique o endpoint /status para detalhes dos erros.
Se o /status retornar erros para alguns anos mas não todos, a API ainda funcionará com os anos que foram coletados com sucesso.
